# Android 懸浮窗模式

實際在其他 App 上方使用時，請使用 Android 原生版的懸浮窗模式。

## 使用流程

1. 打開「點按翻譯助手」。
2. 選擇來源語言與目標語言。
3. 如需直播字幕尾碼，填入翻譯尾碼，例如 `(13`，並勾選「尾碼前自動加入空格」；輸出會是 `翻譯結果 (13`。
4. 如需多國語翻譯，填入翻譯後端，例如 `https://你的後端/v1/translate`。
5. 點「儲存翻譯設定」。
6. 點「啟動懸浮窗」，按系統提示允許顯示在其他應用程式上層。
7. 收起懸浮窗後，在小輸入列輸入文字並按確認。
8. App 會翻譯、附加尾碼、複製到剪貼簿並清空輸入框。
9. 如需點按輔助，先到系統「無障礙」→「已安裝的應用程式」開啟「點按翻譯助手點按輔助」。
10. 拖曳綠色準心到目標位置，收起後按綠色開始鍵，每 400ms 點按一次，最多 90 秒。
11. 點紅色停止鍵、進入文字輸入狀態，或時間到時，點按會停止。
12. 使用者自行在目標 App 貼上與送出。

## 無障礙被系統拒絕

Android 13+ 或 Samsung One UI 可能會限制側載 APK 使用無障礙服務。若看到「系統已拒絕將存取權授予應用程式」或類似提示：

1. 在 App 主畫面點「開啟系統應用程式列表」，或手動打開手機「設定」。
2. 進入「應用程式」清單。
3. 選擇「點按翻譯助手」。
4. 點右上角「⋮」。
5. 選擇「允許受限制設定」並依系統要求驗證。
6. 回到「無障礙」→「已安裝的應用程式」再開啟「點按翻譯助手點按輔助」。

## 翻譯支援

- 繁體中文 ↔ 簡體中文：本機離線處理。
- 繁體中文 ↔ 英文、繁體中文 ↔ 日文、日文 ↔ 英文等：需要雲端翻譯後端。

## 權限說明

- `SYSTEM_ALERT_WINDOW`：顯示懸浮窗。
- `FOREGROUND_SERVICE`：維持懸浮窗服務運行。
- `POST_NOTIFICATIONS`：Android 13+ 顯示前景服務通知。
- `INTERNET`：呼叫使用者設定的翻譯後端。
- `BIND_ACCESSIBILITY_SERVICE`：使用者手動開啟後，執行可見且限時的點按輔助。

本版本不需要麥克風、螢幕錄製或 OCR 權限。

## 合規邊界

懸浮窗提供「翻譯、複製與限時點按輔助」。它不會：

- 自動貼上或自動送出。
- 在未開啟無障礙服務或未按開始鍵時點按。
- 隱藏執行點按。
- 讀取第三方 App 的文字內容。
- 規避平台規則或偵測。
